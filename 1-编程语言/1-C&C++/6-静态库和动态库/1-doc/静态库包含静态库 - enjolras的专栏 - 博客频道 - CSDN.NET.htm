静态库包含静态库

分类： C/C++ Linux 2012-02-29 22:53 575人阅读 评论(0) 收藏 举报
objectmakefilereference测试工作function

都知道静态库是不能包含动态库的，那么静态库能不能包含静态库呢？

以静态库的本质来说，理应是可以的，经过一番试验之后确定可行：


第一步：创建静态库 libother.a

other.h：

[cpp] view plaincopyprint?
  1. #ifndef OTHER_H  
  2. #define OTHER_H  
  3.   
  4. extern "C"  
  5. {  
  6.     void vFuncOther();  
  7. }  
  8. #endif  
#ifndef OTHER_H
#define OTHER_H

extern "C"
{
	void vFuncOther();
}
#endif
other.cpp：

[cpp] view plaincopyprint?
  1. #include "other.h"  
  2. #include <iostream>  
  3. using namespace std;  
  4.   
  5. void vFuncOther()  
  6. {  
  7.     cout<<"vFuncOther is call."<<endl;  
  8. }  
#include "other.h"
#include <iostream>
using namespace std;

void vFuncOther()
{
	cout<<"vFuncOther is call."<<endl;
}
Makefile：

[cpp] view plaincopyprint?
  1. object = other.o  
  2.   
  3. .PHONY default :  
  4. default : $(object)  
  5.     ar rsv libother.a $(object)  
  6. other.o : other.cpp  
  7.     $(CXX) $(CXXFLAG) -c other.cpp  
  8.   
  9. .PHONY clean :   
  10. clean :  
  11.     rm $(object)  
object = other.o

.PHONY default :
default : $(object)
	ar rsv libother.a $(object)
other.o : other.cpp
	$(CXX) $(CXXFLAG) -c other.cpp

.PHONY clean : 
clean :
	rm $(object)
输入make得到libother.a静态库文件


第二步：创建包含libother.a的静态库libmy.a

首先将第一步生成的libother.a静态库文件与头文件拷贝至libmy.a的工作目录。

my.h：

[cpp] view plaincopyprint?
  1. #ifndef MY_H  
  2. #define MY_H  
  3.   
  4. extern "C"  
  5. {  
  6.     void vFuncMy();  
  7. }  
  8.   
  9. #endif  
#ifndef MY_H
#define MY_H

extern "C"
{
	void vFuncMy();
}

#endif
my.cpp：

[cpp] view plaincopyprint?
  1. #include "my.h"  
  2. #include "other.h"  
  3. #include <iostream>  
  4. using namespace std;  
  5.   
  6. void vFuncMy()  
  7. {  
  8.     cout<<"vFuncMy will call vFuncOther later."<<endl;  
  9.     vFuncOther();  
  10. }  
#include "my.h"
#include "other.h"
#include <iostream>
using namespace std;

void vFuncMy()
{
	cout<<"vFuncMy will call vFuncOther later."<<endl;
	vFuncOther();
}
Makefile:

[cpp] view plaincopyprint?
  1. object = my.o libother.a  
  2.   
  3. .PHONY default :  
  4. default : $(object)  
  5.     ar rsv libmy.a $(object)  
  6. my.o : my.cpp  
  7.     $(CXX) $(CXXFLAG) -c my.cpp  
  8.   
  9. .PHONY clean :   
  10. clean :  
  11.     rm $(object)  
object = my.o libother.a

.PHONY default :
default : $(object)
	ar rsv libmy.a $(object)
my.o : my.cpp
	$(CXX) $(CXXFLAG) -c my.cpp

.PHONY clean : 
clean :
	rm $(object)
值得注意的是，这个Makefile是错的，但静态库的编译不会有问题。


第三步：用一个控制台程序测试静态库libmy.a
将第二步生成的libmy.a与my.h拷贝至控制台程序工作目录。

main.cpp：

[cpp] view plaincopyprint?
  1. #include "my.h"  
  2.   
  3. int main()  
  4. {  
  5.     vFuncMy();  
  6.     return 0;  
  7. }  
#include "my.h"

int main()
{
	vFuncMy();
	return 0;
}
代码很简单，用g++编译并链接：

[cpp] view plaincopyprint?
  1. [enjolras@enjolras test]$ g++ -o main main.cpp -lmy -L./  
  2. .//libmy.a(my.o): In function `vFuncMy':  
  3. my.cpp:(.text+0x2b): undefined reference to `vFuncOther'  
  4. collect2: ld returned 1 exit status  
[enjolras@enjolras test]$ g++ -o main main.cpp -lmy -L./
.//libmy.a(my.o): In function `vFuncMy':
my.cpp:(.text+0x2b): undefined reference to `vFuncOther'
collect2: ld returned 1 exit status
报undefined reference，也就是说，在libmy.a中找不到vFuncOther。

原因很简单：

[cpp] view plaincopyprint?
  1. [enjolras@enjolras test]$ ar t libmy.a   
  2. my.o  
  3. libother.a  
[enjolras@enjolras test]$ ar t libmy.a 
my.o
libother.a
libother.a中的other.o，被包多了一层，所以函数vFuncOther就找不到了，那解决办法就很简单了。

修改第二步的Makefile：

[cpp] view plaincopyprint?
  1. object = my.o  
  2.   
  3. .PHONY default :  
  4. default : $(object)  
  5.     ar x libother.a  
  6.     ar rsv libmy.a *.o  
  7. my.o : my.cpp  
  8.     $(CXX) $(CXXFLAG) -c my.cpp  
  9.   
  10. .PHONY clean :   
  11. clean :  
  12.     rm $(object)  
object = my.o

.PHONY default :
default : $(object)
	ar x libother.a
	ar rsv libmy.a *.o
my.o : my.cpp
	$(CXX) $(CXXFLAG) -c my.cpp

.PHONY clean : 
clean :
	rm $(object)
重新make之后，替换libmy.a再次测试：

[cpp] view plaincopyprint?
  1. [enjolras@enjolras test]$ g++ -o main main.cpp -lmy -L./  
  2. [enjolras@enjolras test]$ ./main   
  3. vFuncMy will call vFuncOther later.  
  4. vFuncOther is call.  
[enjolras@enjolras test]$ g++ -o main main.cpp -lmy -L./
[enjolras@enjolras test]$ ./main 
vFuncMy will call vFuncOther later.
vFuncOther is call.
done.
